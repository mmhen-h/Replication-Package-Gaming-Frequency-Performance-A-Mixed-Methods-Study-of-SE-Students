# Replication Package
## *Gaming Frequency Performance: A Mixed-Methods Study of Software Engineering Students*
To ensure transparency and reproducibility of this study, a comprehensive replication package has been made available. This repository contains the data and scripts required to reproduce the analysis of the relationship between gaming habits and academic performance in an undergraduate software engineering course.

## Environment Requirements
R: Version 

Python (Code Counting): Version 3.9.12

Python (Frequency Analysis): Version 3.12.12

Python (Random Generator): Version 3.9.12

## Contents
**Quantitative Analysis Scripts:** R scripts used for statistical testing (ANOVA, Chi-square, and Spearman correlation) and the generation of descriptive visualizations.

**Qualitative Analysis Scripts:** Python scripts used for thematic code counting, calculating the frequency of each code for both low- and high-frequency gamers across cohorts, and the random index generator used to select the 25% sample of rows for manual verification.

**Thematic Codebook:** The finalized codebook used for qualitative coding, defining 28 sub-codes across 8 primary themes, along with example quotes.

**Keywords and References Map:** A structured mapping that lists specific keywords and phrases used by Gemini 3 Pro to automate assigning codes.

**Data Files:** Anonymized CSV datasets containing self-reported student gaming habits paired with final grades retreived from course records.

**LLM Automation Protocol:** The prompts used with Gemini 3 Pro for the second phase (deductive coding), including instructions for data cleaning and markdown-based keyword bolding.

**LLM Outputs:** Both the raw CSV outputs and the CSVs after the 25\% random sample audit.

**PDF copy of the *A Gaming* Survey:** This document contains the original questions and multiple-choice options exactly as they appeared to the participants on the Canvas platform.
